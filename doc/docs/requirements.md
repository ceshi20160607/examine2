# 需求与交付范围（v1）

## 1. 产品目标

交付一套 **可登录、可建系统、可配低代码应用、可录数据、可跑流程、可上传与导出** 的平台，支持：

- 后端独立发版（Jar + Flyway）
- 手机端 H5 / 微信小程序构建发版

## 2. 用户角色

| 角色 | 能力 |
|------|------|
| **平台用户** | 注册/登录（API）、创建多个「系统」、进入某一系统后配置与使用 |
| **系统内用户** | 同一平台账号在不同系统内可有不同 RBAC 角色（成员 + 角色 + 菜单权限） |

无单独的「系统成员邀请」流程：能登录的平台用户，仅能在 **自己创建的系统**（或后续扩展的所有权模型）内操作。

## 6. 验收主路径

与 `mobile/uniapp/README.md` 中「上线前最小回归清单」一致：

1. Health → Login → 创建/进入系统 → 工作台  
2. App → Model → Field → Record 增删改查  
3. Dict → 表单字典字段  
4. Flow：模板 → 发布 → 发起 → 待办处理  
5. Upload：上传与列表  
6. Export：模板 → 任务 → 下载  
7. RBAC：角色/菜单/成员/URI 预览  

API 与页面对照见 [mobile-api-coverage.md](./mobile-api-coverage.md)。
