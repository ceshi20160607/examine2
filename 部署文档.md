# examine2 部署文档

本文说明 **后端（Spring Boot）** 与 **手机端（UniApp）** 的编译、运行，以及 **MySQL 库初始化** 与 **基础数据** 的关系。

---

## 一、环境要求

| 组件 | 版本/说明 |
|------|------------|
| JDK | **21**（与 `backend/pom.xml` 中 `java.version` 一致） |
| Maven | 3.8+ |
| MySQL | 8.x，库名建议 `examine`，字符集 **utf8mb4** |
| Redis | 用于会话等（见 `application.yml`） |
| 手机端 | Node 18+，建议使用 **corepack + pnpm**（与 `mobile/uniapp` 一致） |

---

## 二、数据库：结构（DDL）与迁移

### 2.1 推荐方式：Flyway 自动执行（与线上一致）

- 迁移脚本位置：`backend/examine-web/src/main/resources/db/migration/`
- 应用启动时 **默认开启 Flyway**（`spring.flyway.enabled: true`），会按 `V1`…`Vn` 顺序执行 **建表 + 基础种子数据**（见各 `V*__*.sql`）。
- 业务侧只需先准备好 **空库** 和连接串，**无需**手工跑 DDL（除非 DBA 禁止应用改库表）。

在 MySQL 中创建库（示例）：

```sql
CREATE DATABASE IF NOT EXISTS examine
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_0900_ai_ci;
```

### 2.2 应用数据源与 Redis

默认配置见：`backend/examine-web/src/main/resources/application.yml`（`dev` profile）：

- 端口：`server.port: 9999`
- JDBC：`jdbc:mysql://127.0.0.1:3306/examine?...`
- Redis：`127.0.0.1:6379`（密码示例为 `123456`）

**生产环境**请通过环境变量或独立 `application-prod.yml` 覆盖账号密码，不要将生产密码提交到仓库。

可覆盖的典型项（名称以实际 Spring Boot 外化配置为准）：

- `SPRING_DATASOURCE_URL` / `SPRING_DATASOURCE_USERNAME` / `SPRING_DATASOURCE_PASSWORD`
- `SPRING_DATA_REDIS_HOST` / `SPRING_DATA_REDIS_PORT` / `SPRING_DATA_REDIS_PASSWORD`

## 三、基础数据（Seed）说明

### 3.1 与 Flyway 的关系

以下迁移文件包含**平台/上传/低代码/流程/应用/平台 RBAC** 等基础数据，**随应用启动自动执行**（首次部署即可）：

- `V2__platform_seed.sql` — 平台/默认租户
- `V4__upload_seed.sql` — 本地存储配置示例
- `V6__module_seed.sql` — 演示 app / 模型 / 字段等
- `V8__flow_seed.sql` — 演示流程模板
- `V10__app_seed.sql` — 演示 app client
- `V12__plat_rbac_seed.sql`、`V13__plat_rbac_backfill_account_role.sql` — 平台角色菜单与账号角色回填

### 3.2 手工“仅导入基础数据”的 SQL 文件

仓库根目录提供 **合并版** 脚本（内容等价于以上种子迁移之和，便于 DBA 审计或灾备后补数据）：

- 文件：`基础数据导入.sql`

**使用前提**：

- 表结构已存在且与当前代码版本一致（通常表示 **已跑过对应 DDL 的 Flyway 迁移**，或等价手工建表）。
- 若全新库 **尚未执行过任何迁移**，应优先启动应用让 **Flyway 一次跑全量**，不要只执行本 SQL（会缺表）。

**说明**：种子中的上传本地路径等仅为示例，部署 Linux 时请在库表 `un_upload_storage_config` 中按环境修改为实际目录或对象存储配置。

## 四、后端：编译与运行

在仓库根目录的 `backend` 下执行：

```powershell
cd backend
mvn -q -DskipTests clean package
```

可执行包路径（多模块 Reactor 打包后）：

- `backend/examine-web/target/examine-web-0.0.1-SNAPSHOT.jar`（以实际 `target` 下文件名为准）

运行（示例，指定 `dev` 与配置）：

```powershell
java -jar examine-web\target\examine-web-0.0.1-SNAPSHOT.jar --spring.profiles.active=dev
```

启动成功后：

- 健康/探针：Actuator（已暴露 `health`、`prometheus` 等，见 `application.yml`）
- API 文档：Knife4j（`knife4j.enable: true`）— 浏览器访问以实际根路径为准（通常由 SpringDoc/Knife4j 提供 UI）
- 服务端口：**9999**（`server.port`）

## 五、手机端（UniApp）：安装依赖与发布构建

工程目录：`mobile/uniapp/`

```powershell
cd mobile/uniapp
corepack pnpm install
corepack pnpm run type-check
```

### 5.1 联调地址

`mobile/uniapp/src/config/env.ts` 中 `dev` 环境默认基址为 `http://127.0.0.1:9999`，与后端 `application.yml` 端口一致。若生产域名不同，请改 `test`/`prod` 或运行时切换 `env`。

### 5.2 H5 生产包

需已安装 `sass`（`devDependencies` 已声明，见 `package.json`）：

```powershell
corepack pnpm run build:h5
```

### 5.3 微信小程序

```powershell
corepack pnpm run build:mp-weixin
```

产物在 `dist/build/mp-weixin`，用微信开发者工具导入该目录即可。

> Android/iOS 原生壳工程若由 HBuilderX/云打包生成，请按 DCloud/团队 CI 规范执行，不在本仓库 `pnpm` 脚本中统一体现。

## 六、部署检查清单（建议）

1. MySQL 库可连，时区/字符集正确。
2. Redis 可连，与业务会话一致。
3. 应用首次启动，日志中 Flyway 无失败；表与 `flyway_schema_history` 存在。
4. 访问 `GET /v1/ping`（或项目约定的健康检查）返回成功。
5. 手机端 `env` 指向正确后端地址，HTTPS/小程序合法域名已配置（若适用）。

## 七、常见问题

- **只想要“基础数据 SQL”**  
  用根目录 `基础数据导入.sql`；全新库仍建议以 Flyway 全量为准。

- **上传目录在 Linux 上不可用**  
  修改表 `un_upload_storage_config` 的 `local_root_path` 及 `examine.upload.local-root-path` 配置，保证进程有读写权限。

- **Flyway 与手工 SQL 重复执行**  
  种子多用 `INSERT IGNORE` 或 `ON DUPLICATE KEY UPDATE`，但仍有删除/全量重跑风险，生产环境以变更评审为准。
