## Empty And Error States

| Scenario | Condition | Display content | requestId display | Retry action |
| --- | --- | --- | --- | --- |
| System list empty | PLAT-003 returns `records.length === 0` | Empty systems state; create button follows permission action | Not required | Re-query or create system |
| Account list empty | PLAT-006 returns no records | Empty account state | Not required | Re-query or create account |
| Role list empty | PLAT-009 returns no records | Empty role state | Not required | Re-query or create role |
| Config list empty | PLAT-011 returns no records | Empty config state | Not required | Re-query |
| Detail not found | Detail API returns null/undefined or throws not-found error | Detail drawer empty/error state | Required when API error | Re-query detail |
| API error | `ApiClientError` captured by `errorStore.capture()` | `message`, `code`, `requestId`, retryable flag | Required | Retry when `retryable=true` |
| Local permission denied | `AvailableAction.enabled=false` | `disabledReason` / `stateReason` | Not required until backend fallback error | Refresh session or request permission |

## requestId And Audit

- All page requests pass `authStore.toApiContext(generatedRequestId)` to the typed API client.
- All API error states are normalized through `errorStore.capture()` and retain `requestId`.
- PLAT-002 records `X-Idempotency-Key`; returned `meta.idempotencyReplay` is exposed by `PlatformMutationResult`.
- Platform config values use `redactPlatformConfigValue()` so sensitive config values are not shown in clear text.
- Platform administrators are not given any entry that bypasses system-level permissions to edit business data.
