## 后续测试设计输入

- 端到端主链路：注册/登录 -> 创建系统 -> 初始化默认配置 -> 创建应用和模块 -> 配置字段/页面/菜单/权限 -> 发布 -> 普通用户填报 -> 发起审批 -> 审批处理 -> 导出 -> OpenAPI 调用 -> 审计追踪。
- 权限用例：覆盖平台管理员、系统拥有者、系统管理员、应用管理员、普通用户、审批人、运维人员、外部系统，并按菜单、按钮、字段、数据范围、导出、OpenAPI scope 分层验证。
- 数据隔离用例：覆盖跨系统、跨租户、跨应用、跨模块、跨角色数据范围和字段权限越权。
- 状态流转用例：覆盖系统启停、应用/模块草稿和发布、业务记录草稿/审批中/通过/拒绝/撤回/删除/归档、流程任务待处理/已处理/已取消/已转交/已退回、文件临时/已引用/已删除、导出任务排队/处理中/成功/失败。
- 异常与边界用例：覆盖参数缺失、编码重复、字段类型非法、发布校验失败、无权限、数据不存在、版本冲突、流程模板未发布、候选人为空、文件存储不可用、OpenAPI 签名失败、限流、幂等冲突。
- 并发用例：覆盖自动编号、业务记录重复提交、审批任务重复处理、OpenAPI 幂等重放、导出任务重复创建和任务领取。
- 回归范围：旧项目能力中的登录、系统/租户、动态模块、EAV 记录、权限、流程、上传、导出、OpenAPI、requestId 和日志审计需要作为重构回归基线，但不沿用旧调试式 CRUD 接口作为验收契约。
