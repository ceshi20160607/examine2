# DBA 新 PRD 复审

## 审查结论

**结论：fail**

本次仅复核 `docs/project_understanding.md` 中 `PU-001` 到 `PU-018` 的 DBA 相关关闭条件是否已被新 `docs/prd.md` 满足。新 PRD 已覆盖表前缀、平台账号与系统成员边界、OpenAPI 表域、配置版本、EAV 产品边界、状态矩阵、权限矩阵、文件/导出/OpenAPI 基础语义和 `examine-generator` 落位，但仍有 DBA 进入数据库设计前必须补齐的问题。

fail 的核心原因：

1. `PU-018` 明确要求 analyst 补充旧表“可参考、需重命名、禁止沿用”分类、旧 `un_app_` 历史原因和 flow 实体与 DDL 不一致说明；新 PRD 自身仍声明该项“不阻塞 PRD，但阻塞 DB 设计”，关闭条件未满足。
2. `PU-017` 的初始化数据只停留在方向描述，尚不足以支撑 `sql/init.sql` 的默认账号、平台角色/菜单、默认系统/租户/应用/字段类型元数据等 seed 数据设计。

在以下 issue 关闭前，不建议进入 DB 设计与 SQL 初始化阶段。

## DBA 复核结论

新 PRD 已达到 PRD 级业务边界复核的大部分要求，但 `PU-018` 和 `PU-017` 仍影响数据库设计和初始化 SQL 的可落地性。建议退回 analyst/PM 补齐上述两项后，再由 DBA 复核；复核通过前不要进入 API 契约冻结、DB 设计或 SQL 生成。
