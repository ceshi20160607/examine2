# 前端需求理解审查

## 审查结论

**结论：fail**

analyst 已识别出平台中心、系统管理中心、应用配置中心、应用运行台、流程工作台、文件/导入导出、OpenAPI、运维审计等核心域，也明确旧项目没有可复用前端源码，不能照搬调试式 CRUD 页面。这些方向可以支撑 PM 继续展开 PRD。

但当前需求理解仍不足以直接支撑后续 PRD/API/页面设计。主要缺口集中在页面级闭环、字段级交互、状态矩阵、错误/空态、权限禁用态、保存刷新规则、移动端边界和 typed SDK 契约前置要求。必须由 analyst 或 PM 补充后再进入后续冻结阶段。

## 主要风险

1. **文档可读性存在阻塞风险**
   - `docs/requirement_analysis.md`、`docs/legacy_reference.md`、`docs/service_info.md` 和当前 `docs/prd.md` 在读取时呈现明显中文编码异常，虽然部分语义可推断，但不适合作为后续 PRD、API、页面设计的稳定输入。
   - 需要 analyst 先统一为可直接阅读的 UTF-8 中文文档，避免后续角色误判字段、状态、错误码和页面文案。

2. **页面入口和导航层级仍停留在区域划分**
   - 已区分平台中心、应用配置中心、运行台、流程工作台、运维中心，但缺少可落地的信息架构：登录后默认落点、我的系统、系统切换、租户切换、应用配置入口、应用运行入口、流程入口、运维入口之间的路由层级和跳转规则。
   - 需要 PM 补充角色到入口的映射，包含平台管理员、系统拥有者、系统管理员、应用管理员、普通用户、审批人、运维人员、移动端用户分别看到的一级导航、二级菜单、默认工作台和无权限直达处理。

3. **角色工作台没有形成页面闭环**
   - analyst 提到了工作台、待办、最近访问、健康提醒、导出任务等能力，但未定义各角色工作台卡片、列表、快捷动作、消息入口和数据来源。
   - 需要 PM 补充每类工作台的区块：平台工作台、系统管理工作台、应用配置概览、运行台首页、流程工作台、运维工作台、移动端工作台。

4. **列表字段、筛选和表格行为不够明确**
   - 当前仅说明列表需要分页、筛选、排序、列显示、批量操作、导出等能力，未按页面列出默认列、搜索项、高级筛选字段、批量按钮、行操作、排序字段、状态标签和空列表提示。
   - 需要 PM 至少补齐平台系统列表、租户列表、账号列表、应用列表、模块列表、字段列表、字典列表、角色列表、流程模板列表、待办列表、文件列表、导出任务列表、OpenAPI 客户端列表、调用日志列表、业务记录动态列表的字段矩阵。

5. **表单字段和详情区块缺少页面级定义**
   - 当前已列出字段配置能力和动态表单方向，但缺少管理类固定页面的表单字段，以及动态模块表单 schema 对不同字段类型的 UI 控件、校验提示、默认值、只读、隐藏、联动、子表、附件、关联选择器的行为要求。
   - 需要 PM 补充创建/编辑系统、租户、应用、模块、字段、页面、菜单、角色、流程模板、OpenAPI 客户端、文件存储配置等表单字段；同时补充业务详情页固定区块：基础信息、字段详情、子表、关联数据、附件、流程状态、审批历史、操作记录、评论备注。

6. **按钮动作和状态流转矩阵不足**
   - 已覆盖创建、编辑、启停、删除、发布、审批、导出等动作，但未按对象状态列出按钮可见/禁用/二次确认/成功后状态变化。
   - 需要 PM 补充系统、租户、应用、模块、字段、页面配置、流程模板、流程实例、业务记录、文件、导出任务、OpenAPI 客户端的状态枚举和允许操作矩阵。例如草稿、已发布、启用、停用、归档、审批中、已通过、已拒绝、已撤回、任务生成中、生成失败等。

7. **空态、加载态、错误态和权限禁用态仍是泛化要求**
   - analyst 已指出无系统、无应用、无模块、无字段、无流程、无权限、系统停用、租户未选等状态需要交互设计，但没有形成页面级要求。
   - 需要 PM 补充每类状态的展示策略：页面空态、表格空态、配置未发布、接口加载失败、权限不足、按钮禁用原因、系统/应用/模块停用、记录被流程锁定、文件上传失败、导出失败、OpenAPI 凭证只展示一次、requestId 展示与复制。

8. **保存刷新和脏数据处理规则不明确**
   - 当前仅提到保存成功后列表/详情/表单要同步刷新，但未定义新增、编辑、发布、审批、上传、导出任务创建后的跳转与刷新策略。
   - 需要 PM 明确保存后留在表单还是跳详情、是否刷新列表当前页、详情是否重新拉取、重复提交按钮如何禁用、离开未保存表单如何提示、schema 版本变更时如何重新加载、审批处理成功后待办列表如何移除或刷新。

9. **移动端边界不足以指导设计**
   - 当前只说明移动端包含登录、工作台、应用入口、业务记录查看/轻量编辑、待办处理和消息提醒，不包含复杂配置设计器、批量导入导出和高级运维页面。
   - 需要 PM 补充移动端页面清单、表格替代形态、审批操作区、附件拍照/扫码上传、字段控件适配、不可用功能提示、与 Web 端权限/菜单/schema 的复用边界。

10. **前端 typed SDK 需求需要前置成 API 契约输入**
    - analyst 已提出必须基于冻结 API 建 typed SDK，但还缺少 SDK 生成所需的契约粒度。
    - 需要 PM/API 阶段明确统一响应结构、分页结构、排序/筛选 DSL、上下文字段自动补齐规则、动态字段 typed value 结构、字段类型枚举、权限动作枚举、状态枚举、错误码、requestId、文件上传/下载响应、导出任务状态、审批动作入参、OpenAPI 管理页字段。
