## 发现并修复的问题

1. 系统内“租户启用”失败：页面点击 `SYS-006` 启用已停用租户时，后端返回 403 `SYS_TENANT_DISABLED`。
2. 原因：后端 `changeTenantStatus` 使用了 `activeTenant`，该方法会拒绝停用租户，导致租户一旦停用就无法重新启用。
3. 修复：`changeTenantStatus` 改为按未删除租户查询，不要求当前状态必须是 `ENABLED`；其它必须使用启用租户的业务入口仍保留 `activeTenant` 校验。
4. 回归：重新打包并启动后端后，通过页面再次点击租户启用，`SYS-006` 返回 200。
