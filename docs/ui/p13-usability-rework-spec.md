# P13 可用性返工设计修订

## 背景

用户在部署地址 `http://192.168.0.211:19999/` 试用后反馈：系统无法让普通人顺利使用。PM、Analyst、UI/UX、Frontend 复盘后确认，P12 将构建通过、管理员 E2E 和打包完成误判为“完整普通用户可用”，验收口径需要撤回并进入 P13 返工。

## P0 修订目标

- 登录后必须能从“我的系统”明确创建或进入系统。
- 创建系统后默认进入“系统总览”，不得把新用户直接带到空运行台。
- `SYS-001` 返回的系统、租户、成员和权限上下文必须完整写入前端；系统创建人进入系统后应能创建应用、模块、字段和发布模块。
- 系统内侧栏不得出现无效 `current` 路由，不得混入平台级“平台审计/运维”入口。
- 表单不得预填 P10/P12 测试数据；普通用户看到的是空表单或真实业务数据。
- 成功提示使用中文业务语言，`requestId` 只作为辅助排障信息，不把 `COMMON_OK` 作为主提示。
- 生产环境不执行 `smokeApi` 查询参数入口，避免把接口调试能力暴露给普通用户链路。

## P13 验收主链路

1. 打开前端并登录 `platform_admin / 123123aa`。
2. 在“我的系统”创建系统并进入“系统总览”。
3. 进入“应用”，创建应用。
4. 进入“模块”，创建模块。
5. 进入“字段”，创建字段。
6. 进入“页面配置”，保存默认页面、保存菜单动作、发布检查、发布模块。
7. 进入“运行台”，打开已发布模块并新增记录。

## 本轮实现约束

- 不修改冻结 API、数据库结构和后端业务语义。
- 不生成新业务模块，不扩大 P13 范围。
- 前端只修入口、上下文、权限、导航、提示和普通用户主链路。
- 最终包必须在 TEST-011、VAL-009、REV-009 均通过后由 PKG-002 生成。
