# P13 可用性差异原型

## 1. 我的系统

布局：左侧主面板顶部放“创建一个业务系统”表单，下面是系统列表；右侧为“下一步”提示。

状态：

- 空态：直接创建系统。
- 成功：进入系统总览。
- 禁用：按钮按创建权限禁用，不混用平台中心页面权限。

## 2. 系统总览

布局：作为新系统第一屏，展示配置进度、推荐路径和快捷入口。

关键变化：创建系统不再进入空运行台，避免普通用户面对“暂无菜单”后迷路。

## 3. 应用配置

布局：应用、模块、字段、页面与发布四步工作台。

关键变化：

- 应用、模块、字段表单默认空值。
- 未选应用时“模块”步骤禁用。
- 未选模块时“字段/页面与发布”步骤禁用。
- 不生成 `current` 路径。

## 4. 页面与发布

布局：左侧步骤，中间页面配置，右侧发布提示。

状态：

- 保存默认页面后展示配置摘要。
- 保存菜单动作后运行台可显示入口。
- 发布检查通过后才能作为运行台可用模块。

## 5. 运行台

布局：运行台首页展示业务模块入口；运行模块页展示查询、新增、列表和详情。

关键变化：

- “运行 schema”“模块 ID”“加载运行台”等技术文案改为“表单状态”“业务模块”“打开业务表单”。
- 新建记录字段不再自动填入 P10/P12 测试值。
- 无模块时提示“暂无可用业务模块，请联系管理员发布模块”。
- 成功提示为“操作已完成”，请求号在次级信息中显示。

## 6. 验收对应

本差异原型对应 `docs/test_runs/p13-usability-e2e.md` 的浏览器链路：登录、创建系统、应用、模块、字段、页面发布、运行台新增记录。
