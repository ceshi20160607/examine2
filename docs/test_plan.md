### 数据准备原则

1. 创建系统必须通过 `PLAT-002`，断言响应 `initializedObjects` 包含系统、默认租户、创建人成员、`SYS_SUPER_ADMIN`、默认菜单、默认应用和字段类型引用。
2. 应用、模块、字段、页面、流程、导出模板和 OpenAPI 客户端优先通过公开 API 创建。
3. 幂等测试必须使用可重复但互相隔离的 `X-Idempotency-Key`，同 key 同 hash、同 key 不同 hash、处理中重复请求分别准备。
4. 并发测试要为自动编号、审批任务处理、导出任务领取和 OpenAPI 写接口准备并发触发数据。
5. 所有测试对象编码带 `test_` 或 `demo_` 前缀，便于清理，不得与生产 seed 编码混用。

## API 用例

### 通用断言

所有 API 用例必须断言：

- 响应为统一 `ApiResponse<T>` 或同形错误响应。
- `requestId`、`traceId`、`timestamp`、`success`、`code`、`message`、`data`、`meta`、`errors[]` 符合契约。
- 分页接口使用 `PageResult<T>`，`pageNo`、`pageSize`、`total`、`records`、`hasNext` 可断言。
- 错误响应包含模块化错误码、字段级错误明细和 `retryable`。
- 写接口按契约要求处理 `X-Idempotency-Key`。
- 后端补齐系统上下文、租户上下文、成员上下文、审计字段，前端或测试请求不得伪造只读系统字段。

### API 分组用例

| 分组 | 正常用例 | 异常用例 | 权限用例 | 边界/并发/幂等 |
| --- | --- | --- | --- | --- |
| AUTH | 登录、刷新、退出、me | 密码错误、账号停用、token 过期 | 未登录访问内部 API | 连续失败锁定、重复退出。 |
| PLAT | 创建系统、查询我的系统、账号角色管理 | 系统编码重复、初始化失败 | 非平台管理员管理账号 | 创建系统事务回滚、幂等创建。 |
| SYS/MEM/RBAC | 切换系统租户、邀请成员、部门角色授权 | 租户停用、成员不存在、授权对象非法 | 跨系统访问、字段无写权 | 同一账号重复加入系统、权限缓存刷新失败。 |
| DICT | 创建类型和字典项、启停字典项 | 编码重复、引用中删除 | 无字典维护权限 | 缓存版本、停用项历史回显。 |
| APP/MOD/FIELD/UI | 创建应用模块字段、页面配置、发布检查 | 字段类型非法、发布检查失败、唯一冲突 | 非应用配置管理员越权 | 发布版本冲突、启用唯一规则历史冲突。 |
| RUN | 新增、编辑、详情、列表、删除、提交审批 | 必填缺失、类型错误、状态锁定 | 数据范围越权、字段无写权 | 自动编号并发、保存幂等、提交幂等。 |
| FLOW | 发布模板、发起流程、处理待办、撤回 | 无结束路径、重复审批、原因缺失 | 非候选人审批、跨租户审批 | 任务并发处理、claim/unclaim 幂等。 |
| FILE | 上传、绑定、预览、下载、删除 | 类型禁用、大小超限、存储不可用 | 无业务对象权限下载 | 临时文件过期、已引用文件删除限制。 |
| EXP | 创建导出模板、创建任务、查询任务、下载结果、失败重试 | 模板字段非法、生成文件失败 | 无导出权限、无脱敏权限 | 重复创建、任务领取并发、失败重试幂等。 |
| OPM/OPN | 创建客户端、轮换凭证、外部查询/新增/提交审批 | 签名错误、时间过期、nonce 重放、body hash 错 | scope 越界、IP 白名单拒绝、数据范围越界 | 幂等重放、幂等冲突、限流。 |
| AUD/OPS | requestId 查询、操作日志、请求日志、健康检查 | 查询范围过大、健康异常 | 非审计角色访问 | 日志链路完整性、迁移状态异常。 |
| GEN | 查询生成器映射占位、生成任务占位 | DB 不可用、映射缺失 | 非运维角色访问 | 任务状态冲突，仅验证占位边界。 |

### TEST-002 API 契约用例清单

自动化优先级定义：

- P0：主链路、鉴权、权限、状态流转、幂等或 OpenAPI 安全必跑用例，后续 TEST-003/004 执行时优先自动化。
- P1：核心模块异常、边界、分页筛选、缓存刷新、审计回查用例，纳入接口回归自动化。
- P2：增强或占位能力、低频运维边界，用例先固化，待实现后补自动化。

每组 API 自动化均复用通用断言，并额外断言冻结契约中的路径、方法、鉴权方式、权限点、错误码、枚举状态、`errors[]` 字段定位、`requestId` 链路、分页和筛选 DSL。写接口必须覆盖同 key 同 hash 回放、同 key 不同 hash 冲突和处理中重复请求；读接口如无幂等语义，需明确“幂等不适用”，改测分页、排序、筛选、时间范围或并发只读一致性。

## 权限用例

| 角色 | 应允许 | 应禁止 | 关键断言 |
| --- | --- | --- | --- |
| 平台超级管理员 | 管理平台账号、平台角色、系统和平台审计。 | 默认直接修改系统内业务记录。 | 平台权限与系统内权限隔离。 |
| 系统超级管理员 | 管理本系统成员、组织、角色、菜单、应用、模块、流程、OpenAPI、审计。 | 管理其它系统业务数据。 | `systemId + accountId` 成员上下文正确。 |
| 系统管理员 | 在授权范围内管理系统配置。 | 越权修改未授权菜单、字段、数据范围。 | 后端返回 `PERM_DENIED` 或字段权限错误。 |
| 应用配置管理员 | 配置授权应用、模块、字段、页面、导出模板。 | 修改未授权应用和系统级 RBAC。 | 页面禁用态和后端权限拒绝一致。 |
| 业务用户 | 在菜单、按钮、字段、数据范围内新增、编辑、提交、导出。 | 访问无权限菜单、写无权限字段、下载无权限文件。 | 前端不可见不等于通过，必须调用后端负向验证。 |
| 审批人 | 处理分配给自己的待办，查看关联业务数据。 | 处理非候选任务或已处理任务。 | 返回流程权限或状态冲突错误。 |
| 运维审计人员 | 查询授权范围内日志、健康和任务。 | 修改业务数据或配置。 | 审计入口只读。 |
| OpenAPI 客户端 | 在绑定系统、租户、scope、IP、数据范围内调用。 | 使用内部 Bearer 权限或越权 scope。 | 调用日志记录签名、scope、幂等、限流结果。 |

## 业务场景用例

### 主链路 E2E

1. 平台管理员登录。
2. 创建自定义系统，断言默认租户、系统超级管理员、默认菜单、默认应用和字段类型引用初始化成功。
3. 系统超级管理员进入系统，创建部门、成员、角色和权限。
4. 应用配置管理员创建应用、模块、字段、页面、菜单和导出模板。
5. 配置流程模板并发布。
6. 发布模块配置，普通业务用户能看到运行台入口。
7. 业务用户新增记录、上传附件、保存草稿、提交审批。
8. 审批人处理待办，业务记录状态联动为通过或拒绝。
9. 业务用户或授权人员创建导出任务，成功后下载结果文件。
10. 系统管理员创建 OpenAPI 客户端和 scope，外部调用查询记录、创建记录或提交审批。
11. 运维审计人员按 `requestId` 回查操作日志、请求日志、导出任务日志和 OpenAPI 调用日志。

### 配置发布链路

- 字段编码重复、字段类型非法、页面缺失必填字段、流程无结束路径、导出模板字段越权时，发布检查必须失败并给出可定位错误。
- 发布成功后，运行态读取发布版本；草稿修改不得影响已发布运行态。

### 文件与导出链路

- 临时文件上传后未绑定业务对象时按过期规则处理。
- 业务记录保存成功后附件状态转为已引用。
- 导出任务保存筛选条件、字段权限和脱敏快照。
- 导出失败可重试时状态从 `FAILED` 回到 `QUEUED`，不可重试时保留失败原因。

## 异常和边界用例

| 类型 | 场景 | 预期 |
| --- | --- | --- |
| 参数异常 | 必填缺失、类型错误、非法编码、非法枚举。 | 返回 `COMMON_PARAM_INVALID` 或模块错误码，`errors[]` 定位字段。 |
| 资源异常 | 数据不存在、跨系统资源、软删除资源。 | 返回不存在或无权限，不泄露其它系统数据。 |
| 状态冲突 | 停用模块新增、审批中编辑、终态任务再次处理、成功导出任务重试。 | 返回状态冲突错误码，保留 requestId。 |
| 唯一冲突 | 字段唯一、组合唯一、系统/应用/模块/角色编码重复。 | 返回 409，错误明细包含冲突对象。 |
| 权限异常 | 菜单、操作、字段、数据范围、导出、OpenAPI scope 不满足。 | 后端拒绝，写审计或失败日志。 |
| 幂等冲突 | 同 key 不同 hash。 | 返回 409，`retryable=false`。 |
| 幂等处理中 | 同 key 请求仍处理中。 | 返回 423 和 `Retry-After`。 |
| 并发冲突 | 自动编号并发、流程任务并发处理、导出任务并发领取。 | 只允许一个成功，其它返回冲突或幂等结果。 |
| 文件失败 | 类型禁用、大小超限、存储不可用、引用文件删除。 | 返回 UPLOAD 错误码；业务事务不产生悬挂引用。 |
| 导出失败 | 模板字段非法、文件生成失败、存储失败。 | 任务状态为 `FAILED`，记录 `failureReason` 和重试边界。 |
| OpenAPI 安全 | 签名错误、时间戳过期、nonce 重放、body hash 不一致、IP 拒绝。 | 返回 OPENAPI 错误码并写调用日志。 |
| 限流 | 登录、上传、导出、OpenAPI 超限。 | 返回 429 或排队策略，日志可回查。 |
| 健康异常 | DB、Redis、文件存储、密钥或 migration 异常。 | OPS 接口标记异常，不静默通过。 |

## 幂等与并发专项

| 专项 | 触发接口 | 断言 |
| --- | --- | --- |
| 内部幂等同 key 同 hash | 创建系统、发布配置、保存记录、提交审批、导出任务。 | 返回第一次结果快照，`meta.idempotencyReplay=true`。 |
| 内部幂等同 key 不同 hash | 同一写接口变更 body 重放。 | 返回 `COMMON_IDEMPOTENCY_CONFLICT`。 |
| OpenAPI 幂等同 key 同 hash | OPN 写接口。 | 返回第一次结果快照，调用日志记录幂等结果。 |
| OpenAPI 幂等同 key 不同 hash | OPN 写接口 body 改变。 | 返回 `OPENAPI_IDEMPOTENCY_CONFLICT`。 |
| 自动编号并发 | RUN 创建记录。 | 编号唯一且连续策略符合 DB/API 设计，不使用查询最大值加一。 |
| 流程任务并发处理 | FLOW actions、claim、unclaim。 | 单任务只允许一个有效处理结果。 |
| 导出任务领取并发 | 导出 runner 内部领取。 | 一个任务只被一个执行者领取，失败状态可重试。 |
| 权限缓存刷新失败 | RBAC 授权后缓存刷新异常。 | 返回可重试错误或明确降级策略，并写审计。 |

## OpenAPI 专项

| 场景 | 准备 | 断言 |
| --- | --- | --- |
| 正常签名查询 | 启用客户端、合法 scope、合法 IP。 | 查询成功，日志记录 accessKey、scopeResult、signatureResult。 |
| 创建记录 | `record:create` scope 和字段写权限。 | 记录创建成功，内部业务规则和字段权限仍生效。 |
| 提交审批 | `record:submit` scope 和流程绑定。 | 记录进入审批中，流程实例创建。 |
| 处理流程任务 | `flow:task:handle` scope 和外部处理人映射。 | 任务状态变化并写审计。 |
| 下载文件 | `file:download` scope 和文件引用权限。 | 只允许授权文件下载。 |
| 签名失败 | 错误 secret 或 canonical request。 | `OPENAPI_SIGNATURE_INVALID`。 |
| 时间窗口失败 | timestamp 超出窗口。 | `OPENAPI_TIMESTAMP_EXPIRED`。 |
| nonce 重放 | 相同 nonce 重复调用。 | `OPENAPI_NONCE_REPLAY`。 |
| scope 越界 | 未授予对应 scope。 | `OPENAPI_SCOPE_DENIED`。 |
| 数据范围越界 | 记录不在客户端数据范围内。 | `OPENAPI_DATA_SCOPE_DENIED`。 |
| 限流 | 超过 rateLimitPolicy。 | `OPENAPI_RATE_LIMITED` 且 `retryable=true`。 |

## 导出专项

| 场景 | 准备 | 断言 |
| --- | --- | --- |
| 创建导出模板 | 选择允许导出的字段。 | 模板保存成功，字段权限合法。 |
| 创建导出任务 | filters 或 selectedRecordIds、模板、幂等键。 | 任务为 `QUEUED`，保存筛选和权限快照。 |
| 任务成功 | 后续执行 runner 完成。 | 状态为 `SUCCESS`，存在 `resultFileId`，下载复用文件权限。 |
| 任务失败 | 模板字段非法、文件生成失败或存储失败。 | 状态为 `FAILED`，记录 `failureReason`。 |
| 失败重试 | retryable=true。 | 任务重新进入 `QUEUED`。 |
| 无导出权限 | 无 `EXPORT_JOB_CREATE` 或字段脱敏权限。 | 返回 `EXPORT_PERMISSION_DENIED`。 |
| 并发领取 | 多执行者领取同一任务。 | 只有一个领取成功。 |
