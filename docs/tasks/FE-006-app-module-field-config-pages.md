# FE-006 应用模块字段配置页面

- taskId: FE-006
- 标题: 应用模块字段配置页面
- 负责角色: frontend
- 所属大任务/模块: 前端 / 应用配置
- 目标: 实现应用、模块、字段、页面、菜单、动作和发布配置页面。
- 输入文件: `docs/api.md`、`frontend/src/api/`
- 输出文件或输出目录: `frontend/src/pages/module-config/`、`frontend/docs/page-contracts/FE-006-app-module-field-config-pages.md`

## 详细工作内容

- 实现 APP、MOD、FIELD、UI 接口对应配置页面。
- 支持字段类型、必填、默认值、字典、关联、子表、权限、列表/表单/详情 schema 配置。
- 支持发布检查、发布失败定位、发布成功刷新运行菜单和 schema 缓存。
- 补齐页面级 API 映射证据，覆盖路由、API ID、必填入参、响应字段、上下文依赖、枚举/状态/错误码、权限禁用态、空态/错误态、requestId 展示和无旁路请求检查。

## 完成状态定义

- 默认状态: pending。
- 完成条件: 应用配置管理员能完成模块建模并发布，`FE-006` 页面级契约证据文件已补齐。

## 验收标准

- 字段保存后刷新字段列表和页面预览。
- 发布检查错误能定位配置项。
- 权限禁用态、空态、错误态和完成证据可被 FE-012 汇总到 `frontend/docs/api-contract-map.md`。

## 测试/自检要求

- 自检字段编码重复、字段类型切换、页面引用无效字段、发布失败状态。
